Researching the effect of generative and simple data augmentation methods for skin lesion datasets

Data augmentation, Generative AI, Deep learning, Python

The ability to effectively analyse skin lesions opens the door for earlier diagnoses, which leads to better treatment and lower mortality. Deep learning models present a promising avenue toward achieving this critical objective, yet the effectiveness of these models heavily hinges on the presence and quality of datasets.

The ability to effectively analyse skin lesions opens the door for earlier diagnoses, which leads to better treatment and lower mortality. Deep learning models present a promising avenue toward achieving this critical objective, yet the effectiveness of these models heavily hinges on the presence and quality of datasets.

Data augmentation is a technique of artificially increasing the size of a dataset by creating modified copies of a dataset using existing data. This study analyses common data augmentation techniques, such as cropping images or changing their brightness, and newer techniques like generative adversarial networks (GANs) which generate new data samples.

The specific type of GAN used was a conditional generative adversarial network (cGAN). This was used to augment new data for each of the 8 classes of skin lesion in the ISIC 2019 dataset.

This project evaluated the extent that several data augmentation techniques can improve the capability of deep learning models to classify skin lesions. The methods will be applied individually to a skin lesion dataset, then run on multiple different deep learning models, ResNet and DenseNet. Each method will be evaluated on several metrics including sensitivity and AUC. These findings offer a reference point to assist in optimisation of deep learning models used in future research regarding skin lesion classification.

GPU Optimisation for Molecular Dynamics

Microsoft Azure, GPU core optimisation, Linux

In this project I was given 3 versions of a “.c” file that simulate molecular dynamics; the serial version of the code, an initial attempt at parallelisation of the code using OpenMP, and an initial attempt at parallelisation using MPI. The aim of the task was to optimise the individual executables, while ensuring the output stays correct. To compile and run code I was using Microsoft Azure. The approach I took to optimise the files was to initially apply different levels of “-O” optimisations provided by Intel. These all produced impressive results without impacting the accuracy of the output.

On top of this, I updated the OpenMP code through Nano. The original OpenMP code calculated certain values in a shared array; I updated this so that local variables are used and incremented with each thread in order to reduce overhead. I did this using a “reduction” pragma statement, which improved the performance of the code when running across any number of GPU cores. This code was compiled several times using different levels of Intel’s “-O” optimisations.

Using Apache Spark to analyse Transport Data

Apache Spark, Big Data, Python

In this project, I aimed to test the research hypothesis: "Rides in 2014 starting from Aston Street, Stepney station were shorter compared to other stations." To analyse this, I used Apache Spark to process and clean a large dataset from Transport for London comprising multiple `.csv` files. These files were combined into a single data frame, which was then filtered to include only rides from 2014. Key columns such as "Start date," "StartStation name," and "Duration" were prepared for analysis by converting strings into appropriate data types and extracting relevant information.

The data was further divided by month to allow for a more granular analysis. For each month, I calculated metrics such as the mean journey duration from Aston Street and compared it to the mean journey duration from other stations. This comparison was carried out using t-tests from the SciPy package to determine whether the differences were statistically significant.

Finally, I aggregated the results across all months to determine the overall significance, leading to a conclusion about the initial hypothesis. The analysis showed that the hypothesis was false, as rides from Aston Street were not significantly shorter than those from other stations.

Building a KNN algorithm from scratch to classify tumours as benign or malignant

It is no secret about the huge potential of applying AI in medical settings. This is a K-Nearest Neighbours algorithm, which I built from scratch in java. It can accurately classify unknown data points with an accuracy of 96%. The dataset contains information on benign and malignant tumours, with 30 different attributes about their size and texture.

The way the algorithm works is by first shuffling the dataset of over 500 separate entries, then splitting into a training dataset, where the labels are kept, and a testing dataset, where the labels are removed so they can be predicted. Then it plots the training dataset in the sample space (graph). The sample space is of the same dimensions of 30 attributes.

Next, to determine the nature of the unknown testing data, it plots the unknown data in the same sample space as the known training data. By using pythagorean distance, the algorithm calculates the unknown data points’ ‘nearest neighbours’; the labels of the nearest data points in the sample space. Whatever label is most prevalent near each unknown datapoint is the label assigned to the unknown data point, classifying each unknown data point as either showing a benign or malignant tumour.

Deep learning for Lyme disease detection

I have already mentioned how the massive potential of applying AI to medical scenarios. I have also mentioned my plans to rigorously research the effectiveness of several data augmentation methods in the context of skin lesions.

Recently, I made a small scale experiment to test the waters. This one was not specifically to detect benign or malignant skin lesions, but to detect Erytha Migrans, a common manifestation of Lyme disease on the skin. I used a database of over 3,000 images, which I tested on a ResNet50 model and a MobileNetV2 model. This was done using Python in Google Colab.

Unfortunately, the results were skewed. However, after looking into why the models were not performing adequately, I proved that the issue is no doubt due to poor quality of data. The data contains a lot of irrelevant images, meaning that not all 3,000 are of use to the deep learning models and skew the outputs. This only emphasises the work that must be done to increase the availability of high quality skin lesion datasets if valuable research is to be done on implementing deep learning models to this field.

However, the amount of quality data is slim, which is why my research into the merit of individual data augmentation techniques is important.
